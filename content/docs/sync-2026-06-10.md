# Sync 2026-06-10 — skills 구조 정리

회사 PC에서 2026-06-10 수행한 구조 정리의 기록. 개인 PC에 동일하게 재적용하여 두 머신의 `~/.claude/` 사본을 동기화한다.

## 배경

Claude Code는 `skills/<name>/SKILL.md` 1-depth만 스캔한다. `skills/_personal/` 하위 skill 3개는 전부 휴면 상태였다 (직하 이동 후 세션 skill 목록에 나타나는 것을 실증 확인). `ROLE_TUTOR.md`는 어디서도 참조되지 않는 고아 파일로 삭제 확정.

## 변경 내역

### 삭제

| 대상 | 사유 |
|---|---|
| `roles/ROLE_TUTOR.md` | 어디서도 참조되지 않는 고아 파일 |
| `skills/_personal/chat-code-handoff/` | Two-CLI HANDOFF.md 템플릿 + ROLE_BUILDER 규약에 완전 대체됨 |
| `skills/_personal/gate-checkpoint/` | CLAUDE.md §6 self-review + verification-loop skill에 흡수됨 |

### 이동

| 원위치 | 새 위치 | 사유 |
|---|---|---|
| `skills/_personal/ue-umg-review/` | `skills/ue-umg-review/` | 1-depth 스캔 영역으로 직하 이동 (휴면 해제) |
| `skills/_personal/` (빈 디렉토리) | 삭제 | 잔여물 정리 |

### README.md 수정 (3곳)

| 위치 | 변경 |
|---|---|
| 구조 트리 | `_personal/` 하위 3개 블록 → `ue-umg-review/` 직하 1줄 + 1-depth 스캔 주석 |
| Layer 표 (L5) | `skills/_personal/` → `skills/ue-umg-review/` 등 자작 skill (skills/ 직하) |
| "진화" 절 | 새 도메인 진입 시 `_personal/`이 아닌 `skills/` 직하에 skill 시작 (하위 폴더 미스캔 명시) |
